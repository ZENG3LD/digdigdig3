## What dig3 is today

A Rust connector library for **47 exchanges** (CEX + DEX + forex + stocks + predictions + data providers), exposed through a single `ExchangeHub` async pool:

```rust
let hub = ExchangeHub::new();
hub.connect_full(ExchangeId::Binance, &[AccountType::Spot, AccountType::FuturesCross], false).await?;
let rest = hub.rest(ExchangeId::Binance).unwrap();
let ws   = hub.ws(ExchangeId::Binance, AccountType::FuturesCross).unwrap();
let ticker = rest.get_ticker("BTCUSDT".into(), AccountType::Spot).await?;
let stream = ws.event_stream();
```

Other CryptoCompare cells (orderbook UnsupportedOp, exch_info rate-limit, WS_trade/kline silent) are wire-not-present or quiet-channel.

- **Dukascopy** — no public live REST endpoints (tick data only).

## Architecture summary

### Layer 1 (existing, stable): `digdigdig3` crate

- `ExchangeHub` — async pool, `Arc<dyn CoreConnector>` vtable dispatch
- `UniversalWsTransport<P: WsProtocol>` — generic WS transport with reconnect/ping/sub-replay/dispatch_all multi-parser fan-out
- 47 connectors under `src/l{0,1,2,3}/{open,paid,gated}/...`
- `SymbolNormalizer` (22 sub-modules) — canonical ↔ raw symbol per exchange
- `ConnectorCapabilities` (71 flags) + `CapabilityProvider::supports(kind, account)` derived from `topic_registry`
- Naming convention: `ExchangeError::UnsupportedOperation` = TODO, `ExchangeError::NotSupported` = wire-not-present (with citation)

### Layer 2 (planned, not yet built): `dig3-station` crate

See `docs/plans/station-architecture.md` for the full design. TL;DR:

```rust
let station = Station::builder()
    .persistence(Persistence::on().bars(true).trades(true)...)
    .cache(Cache::on().warm_start(WarmStart::LastN(300))...)
    .reconnect(Reconnect::on().gap_heal(GapHeal::OnReconnect))
    .multiplex(Multiplex::on().deferred_unsub(Duration::from_secs(30)))
    .build().await?;

let handle = station.subscribe(
    SubscriptionSet::new()
        .add(Binance, "BTC-USDT", FuturesCross, [Trade, Orderbook, MarkPrice])
).await?;

while let Some(event) = handle.recv().await { ... }
drop(handle); // RAII deferred-unsub
```

Designed to borrow MLC's strong patterns (SharedMap dual-read, deferred-unsub, sequenced REST→WS) and fix its weak ones (try_send drop, dual independent refcount drift).

## What's next (Phase 1 of Station)

Detailed plan in `docs/plans/station-phase-1-plan.md`. Eight steps:

1. **Workspace split**: move `src/` → `crates/dig3-core/src/`. Create `crates/dig3-station/` and `crates/dig3-cli/`. Top-level `[workspace]` Cargo.toml.
2. **Station skeleton**: empty lib + Cargo with feature flags (`persistence` / `cache` / `multiplex` / `reconnect` / `orderbook-tracker` / `bar-loader` / `metrics` / `prometheus`).
3. **SubscriptionSet + SubscriptionHandle**: declarative API + RAII Drop unsubscribe.
4. **TradeWriter**: binary append-only `[u64 ts_ns, f64 px, f64 qty, u8 side, u64 tid]` = 41 bytes/record + sparse `.idx`.
5. **RestCache**: LRU + TTL for `get_ticker`.
6. **Station::builder + .subscribe()**: builder constructs StationConfig, build() returns Station, subscribe lazy-spawns multiplexer actors.
7. **dig3-cli `watch trades`**: end-to-end CLI command.
8. **Verify + commit**: workspace cargo check clean, 821 tests, TRUSTED still 18.

Acceptance: `target/release/dig3 watch trades binance btc-usdt --duration 30` prints live trades AND persists to `./dig3_storage/trades/binance/spot/btcusdt/2026-05-20.dat`.

## Open architectural questions deferred to later phases

1. **Persistence atomicity on crash**: CRC32 per-record vs fsync per-record. Lean CRC.
2. **SharedMap memory pressure**: 10k × 5 tf × 50 sym × 18 exch ≈ 2GB worst case. Phase 2-3: LRU eviction at series level, warm-load from disk on next access.
3. **Subscription update semantics**: `handle.add(...)` immediate; `handle.remove(...)` deferred (consistent with builder default).
4. **Stream type explosion**: 9 × 18 × 3 = 486 potential actors; lazy spawn only when sub'd.
5. **`metrics` crate version**: pin 0.23, exporter-prometheus 0.15; revisit if upstream breaks.
